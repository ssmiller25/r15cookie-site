Blogging is difficult. Long form articles take time to compose. As I reviewed this site, most of the updates were smaller updates to the public knowledge that I felt inspired to keep. So why not lean into that? Thus the rebuild of this site. 

Now all content-related commits are surfaced on the homepage. Longform blog post are still represented. But now smaller updates can also carry meta-content. I ran into a small issue as git commits/history are immutable. Typos cannot be fixed without rewriting git history. So I browered a feature that I haven't used myself in the past: [git notes](https://git-scm.com/docs/git-notes). Thanks you AI/LLMs for the suggestion!

We'll try this new methodology and see were it goes. With everything in markdown and git, future pivots are easy.
